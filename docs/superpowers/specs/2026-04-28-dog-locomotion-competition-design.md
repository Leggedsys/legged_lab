                         │ [vx, vy, ω, h_cmd] 或切换Tier2

## 3. Tier 1：通用运动策略

### 3.1 网络架构（非对称 Actor-Critic）

**Actor 输入（~424 维）：**
```
base_lin_vel          3    （机体系线速度，含噪声）
base_ang_vel          3    （机体系角速度，含噪声）
projected_gravity     3    （投影重力方向，含噪声）
commands              4    （vx, vy, ω, h_cmd）
joint_pos_rel         12   （关节位置相对默认值）
joint_vel_rel         12   （关节速度）
last_action           12   （上一步动作）
height_scan (噪声)    375  （25×15 网格，见3.3节）
```

**Critic 额外特权输入（~392 维，仅训练时）：**
```
exact_height_scan     375  （无噪声精确高度图，替换 Actor 的噪声版）
contact_forces        12   （4脚×3轴接触力）
foot_friction         4    （各脚摩擦系数）
mass_offset           1    （质量扰动量）
```

**网络结构：**
```
Actor MLP:  [512, 256, 128]，激活函数 ELU
Critic MLP: [512, 256, 128]，激活函数 ELU
            输入 = Actor输入 + 特权观测
```

Actor 的高度扫描在训练时施加噪声（±2cm + 20% 随机 dropout），无需额外蒸馏，直接对齐深度相机的实际误差特性。

### 3.2 指令空间

| 指令 | 范围 | 说明 |
|------|------|------|
| vx | [-0.5, 1.0] m/s | 前进/后退 |
| vy | [-0.3, 0.3] m/s | 侧移 |
| ω | [-0.8, 0.8] rad/s | 转向 |
| h_cmd | [0.15, 0.30] m | 目标躯干高度（正常0.28，蹲行0.15） |

`h_cmd = 0.15` 时，机身顶部距地约 0.27m，通过 300mm 限高杆留有约 30mm 安全余量。

### 3.3 高度扫描（Exteroception）

```
网格规格: 25 × 15 = 375 点
覆盖范围: 纵向 1.0m（以机体为中心前后各0.5m）
          横向 0.6m（以机体为中心左右各0.3m）
分辨率:   纵向 ~4.2cm，横向 ~4.3cm
```

20cm 纵板可采到约 5 个点，足够可靠检测板边缘；15cm 横板间隙可采到约 3 个点，能预判步态。

训练时噪声注入：
- 高斯噪声 σ = 1.5cm
- 随机遮挡 dropout 率 20%（模拟深度相机近场盲区和反光干扰）

### 3.4 奖励函数

**主奖励（驱动任务目标）：**

| 项目 | 权重 | 参数 | 说明 |
|------|------|------|------|
| track_lin_vel_xy_exp | 2.0 | std=0.25 | 线速度跟踪 |
| track_ang_vel_z_exp | 0.5 | std=0.25 | 转向跟踪 |
| track_height_exp | 1.5 | std=0.03 | 躯干高度指令跟踪（新增） |
| feet_air_time | 0.3 | threshold=0.5s | 步态激励，防滑步 |

**惩罚项（防止策略退化）：**

| 项目 | 权重 | 说明 |
|------|------|------|
| lin_vel_z_l2 | -0.5 | 抑制垂直振荡 |
| ang_vel_xy_l2 | -0.1 | 抑制翻滚/俯仰扰动 |
| dof_torques_l2 | -1e-5 | 减少电机过载 |
| dof_acc_l2 | -2.5e-7 | 防止关节高频颤振 |
| action_rate_l2 | -0.01 | 平滑动作 |
| flat_orientation_l2 | -0.5 | 姿态稳定（蹲行时自动放宽） |
| foot_slip | -0.1 | 防滑移（砂砾坑/板桥关键） |
| undesired_contacts | -2.0 | calf 接触惩罚 |

### 3.5 域随机化事件

| 事件 | 参数 | 时机 |
|------|------|------|
| randomize_rigid_body_material | 摩擦 0.4–1.2 | startup |
| randomize_rigid_body_mass | ±1.5kg（base_link） | startup |
| randomize_joint_parameters | 刚度±10%，阻尼±10% | startup |
| push_robot | 力 0–25N，间隔 5–10s | reset（每步触发概率） |
| base_external_force_torque | 力0–10N，力矩0–2Nm | reset |

### 3.6 地形课程（4 阶段）

**阶段1（迭代 0–500）：平地 + 域随机化**
- 地形：平地
- 摩擦：0.4–1.2（含低摩擦，覆盖砂砾坑）
- 质量扰动、推力事件启用
- h_cmd 范围全开，策略学会高度控制

**阶段2（迭代 500–1500）：粗糙地形 + 斜坡**
- 随机粗糙地形：高度方差 ≤ 3cm
- 斜坡：坡度 ≤ 15°
- 砂砾坑：局部区域摩擦 0.3–0.5 + 随机小突起（高度 ≤ 1cm）

**阶段3（迭代 1500–3000）：结构化地形**

| 地形 | 课程起点 | 完整难度（比赛参数） |
|------|----------|---------------------|
| 横板桥 | 板宽40cm，间距5cm | 板宽40cm，间距15cm |
| 纵板桥 | 板宽30cm，间距5cm | 板宽20cm，间距10cm |
| 楼梯 | 阶高30mm | 阶高100mm，宽30cm |

**阶段4（迭代 3000–5000）：完整难度 + 混合**
- 全部地形类型随机混合
- 纵板桥全难度（20cm板，10cm间距）
- 推力加强（最大30N）
- h_cmd 在 0.15–0.30 范围全随机采样

### 3.7 PPO 超参数

| 参数 | 值 | 备注 |
|------|----|------|
| num_envs | 4096 | 地形课程需要足够多样性 |

| num_mini_batches | 4 | — |
| gamma | 0.99 | — |
| lam | 0.95 | — |
| init_noise_std | 0.8 | 大初始噪声鼓励早期探索 |
| empirical_normalization | True | — |

## 4. Tier 2：爬墙专项策略

### 4.1 任务分解

```
Phase A: 接近      →  Phase B: 前腿上墙    →  Phase C: 躯干翻越   →  Phase D: 后腿跟进
前腿距墙 ~40cm        前腿抬至 >300mm          重心前移过墙顶          后腿踏上，完成爬越
低速前进              后腿支撑蓄力              允许身体前倾            恢复正常站姿
```

### 4.2 网络架构

```
输入（~62 维）：
  base_lin_vel        3
  base_ang_vel        3
  projected_gravity   3
  joint_pos_rel       12
  joint_vel_rel       12
  last_action         12
  wall_distance       1    （深度相机，0–1m 归一化）
  front_foot_height   2    （FL/FR 脚端 z 高度，相对地面）
  climb_phase         4    （one-hot，A/B/C/D 由 Tier 3 注入）

MLP: [256, 128, 64]，ELU
无高度扫描（墙为硬结构，相机距离足够）
```

### 4.3 奖励函数（阶段感知）

| 阶段 | 奖励项 | 权重 | 条件 |
|------|--------|------|------|
| A（接近） | forward_progress | 1.0 | CoM 向墙推进 |
| A | base_stability | 0.5 | 保持平稳 |
| B（前腿上墙） | front_foot_height | 3.0 | 前脚 z > 0.30m |
| B | rear_support | 1.0 | 后腿接触力 > 阈值 |
| B | body_wall_contact | -5.0 | 身体碰墙惩罚 |
| C（躯干翻越） | com_over_wall | 4.0 | CoM x 越过墙顶中线 |
| C | pitch_penalty | -1.0 | 俯仰角过大惩罚 |
| D（后腿跟进） | rear_foot_height | 2.0 | 后脚 z > 0.30m |
| D | full_cross_bonus | 10.0 | 四脚均越过墙顶，一次性 |
| 全局 | alive | 1.0 | — |
| 全局 | joint_torques_l2 | -5e-5 | 防电机过载 |

### 4.4 训练环境与超参数

```
场景: 机器人出生点距墙 0.8–1.2m（随机）
墙高: 280–320mm（±20mm，覆盖制造公差）
地面摩擦: 0.6–1.0

PPO:
  max_iterations: 2000
  num_envs:       1024
  learning_rate:  5e-4
  entropy_coef:   0.02
  init_noise_std: 1.0
```

## 5. Tier 3：任务规划器

### 5.1 感知模块

| 检测目标 | 方法 | 输出 |
|----------|------|------|
| 横板桥 | 点云检测周期性间隙（间距15cm，板宽40cm） | 区域起止、行进方向 |
| 纵板桥 | 检测平行窄平面（宽20cm，间隙10cm） | 板中线坐标、横向偏差 |
| 限高杆 | 检测横向连续点云段，高度250–400mm | 杆距离、杆底高度 |
| 300mm墙 | 检测大面积垂直平面，高度280–350mm | 墙距离 |
| 立柱（绕杆） | 圆柱体拟合 | 各柱 x/y 坐标列表 |
| 砂砾坑 | 局部区域高度方差检测 | 坑起止距离 |

所有检测结果附置信度，低于阈值时保持上一帧（防止策略闪烁切换）。

### 5.2 状态机

```
                    ┌─────────────────────────┐
                    │       FREE_WALK          │  ← 默认，手柄控制
                    └──┬───┬────┬────┬────────┘
                       │   │    │    │
         检测到横/纵板桥│   │    │    │检测到立柱群
                       ▼   │    │    ▼
                ┌────────┐ │    │ ┌────────┐
                │ BRIDGE │ │    │ │ WEAVE  │
                └────────┘ │    │ └────────┘
                           │    │
              检测到300mm限高杆  检测到300mm墙
                           ▼    ▼
                      ┌──────┐ ┌───────┐
                      │CROUCH│ │ CLIMB │ ← 切换Tier2策略
                      └──────┘ └───────┘
```

所有非 FREE_WALK 状态完成后自动回到 FREE_WALK。砂砾坑不切换状态，Tier 1 自行适应（限速至 0.3m/s）。

### 5.3 各状态指令生成

**BRIDGE_HORIZONTAL（横板桥）：**
```
vx = 0.20 m/s（限速，让步态有时间对准间隙相位）
vy = 0（横板桥较宽，无需侧向修正）
ω  = Kp × heading_error（朝向修正）
h_cmd = 0.28
```

**BRIDGE_VERTICAL（纵板桥）：**
```
vx = 0.15 m/s（极低速，侧向精度优先）
vy = Kp × lateral_offset_from_plank_center（视觉闭环对中）
ω  = Kp × heading_error
h_cmd = 0.28
```

**CROUCH（限高杆）：**
```
检测到杆: h_cmd 逐步降至 max(0.15, bar_bottom_height - 0.13)
vx = 0.15 m/s
完全通过后: h_cmd 逐步恢复 0.28，切回 FREE_WALK
```

**WEAVE（绕杆）：**
```
输入: 各柱 x/y 坐标
路径: 几何插值生成绕杆 waypoint 序列
指令: PD 跟踪 waypoint → [vx, vy, ω, 0.28]
```

**CLIMB（爬墙）：**
```
暂停 Tier 1，切换至 Tier 2 策略
注入: wall_distance（实时）、climb_phase（由CoM位置判断）
完成条件: 四脚均越过墙顶 → 恢复 Tier 1 + FREE_WALK
```

### 5.4 接口定义

```
Tier 3 → Tier 1:  [vx, vy, ω, h_cmd]            @50Hz
Tier 3 → Tier 2:  [wall_distance, climb_phase]   @50Hz
Tier 1/2 → 电机:  joint_position_targets (12维)  @50Hz
深度相机 → Tier 3: 点云 + 局部高度图             @30Hz
```

## 6. 训练阶段规划

```
Week 1: Tier 1 阶段1+2（平地到粗糙地形）
         目标: mean_reward 稳定提升，error_vel_xy < 0.2 m/s

Week 2: Tier 1 阶段3（结构化地形，板桥/楼梯引入）
         目标: 100mm楼梯通过率 > 80%，纵板桥不落脚

Week 3: Tier 1 阶段4（完整难度）+ Tier 2 训练（并行）
         目标: 纵板桥通过率 > 70%，爬墙前腿上墙率 > 60%

Week 4: Tier 2 收敛 + Tier 3 感知模块实现 + 系统联调
         目标: 全流程仿真跑通，各任务单独测试通过
```

## 7. 比赛地图与训练场景

**不直接复刻比赛地图**，原因：过拟合固定路线，泛化性差；仿真与实物位置误差导致适得其反。

**正确做法：**
- 提取比赛各地形的**具体参数**（见第1节）放入课程
- 每次训练随机化地形**位置和出现顺序**
- 最终验证阶段用接近比赛布局的固定场景做测试（不参与梯度更新）
