# 训练配置改动记录

## 2026-04-24 | 初训配置修订（DogEnvCfg / DogWalkEnvCfg）

### 背景
首次训练前对基础配置进行系统性审查，发现多处参数设置会阻碍策略收敛或损害 sim-to-real 迁移能力。

---

### Action Scale
- **改动**：`front_legs` / `rear_legs` 所有关节 scale 从 0.02~0.035 统一提升至 0.25~0.35
- **原因**：原值过小，策略输出 [-1, 1] 时关节偏移仅 ±0.02~0.035 rad，无法形成有效步态，训练易陷入微小抖动

### 前后腿 Action Scale 对称化
- **改动**：前后腿 scale 统一为相同值
- **原因**：原来前后腿数值不一致，在无明确机械不对称依据的情况下引入不必要的动力学偏差

### PD 增益统一（DogEnvCfg）
- **改动**：前腿 Kp=16/Kd=0.9，后腿 Kp=22/Kd=1.2 → 统一为 Kp=20/Kd=1.0
- **原因**：前后腿刚度差异约 37%，会导致步态前后不一致；统一后若实测发现需要分开调再基于数据决定

### Domain Randomization 开放
- **改动**：
  - 摩擦系数：(1.0, 1.0) → static (0.6, 1.2) / dynamic (0.5, 1.0)
  - 质量扰动：(0.0, 0.0) → (-1.0, 1.0) kg
  - 外力扰动：(0.0, 0.0) → force (0, 10) N / torque (0, 2) N·m（保守初训值）
- **原因**：完全关闭 DR 导致策略对仿真环境过拟合，是 sim-to-real 失败的主要原因之一

### 初始化范围扩大（EventCfg）
- **改动**：
  - `reset_base` yaw 范围：±0.05 → ±π
  - `reset_robot_joints` position_range：(0.995, 1.005) → (0.9, 1.1)
- **原因**：原来初始化过于集中，策略缺乏姿态鲁棒性；yaw 全范围随机化使策略学会从任意朝向响应速度指令

### Reward 权重调整（DogEnvCfg）
- **改动**：`flat_orientation_l2` weight：-20.0 → -2.0
- **原因**：与 DogWalkEnvCfg 中的 -1.0 差距缩小，避免课程切换时策略剧烈震荡

### 指令空间扩展（DogWalkEnvCfg）
- **改动**：
  - `lin_vel_x`：(0.3, 0.6) → (-0.3, 0.6)，支持后退
  - `lin_vel_y`：(0.0, 0.0) → (-0.2, 0.2)，支持侧移
  - `ang_vel_z`：(0.0, 0.0) → (-0.5, 0.5)，支持转向
  - `resampling_time_range`：(4.0, 4.0) → (4.0, 8.0)，增加指令时长多样性
  - `rel_standing_envs`：0.0 → 0.1，保留 10% 站立环境防止策略退化
- **原因**：原配置只学单向前进，竞赛场景需要全向运动能力

### Reward 补全（DogWalkEnvCfg）
- **改动**：
  - `track_ang_vel_z_exp`：0.0 → 1.0，与转向指令空间对应
  - `feet_air_time`：0.0 → 0.5，激励真实抬腿步态，防止滑步
  - `dof_acc_l2`：0.0 → -2.5e-7，约束关节加速度，防止高频颤振
- **原因**：原配置这三项均为 0，策略无转向能力、易退化为滑步、易出现颤振

---

### 待观察
- 实际训练后若前后腿动力学存在明显差异，根据 episode 内步态对称性指标重新分开调 PD 增益
- 外力扰动范围（当前保守值）可在策略稳定后逐步提升至 (0, 30) N / (0, 5) N·m
- num_envs 当前 128，显存允许时建议提升至 1024 以上加快采样效率
